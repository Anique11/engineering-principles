# Engineering Principles

This repository collects engineering principles that have emerged through designing, building, maintaining, and reflecting on software over the course of my career.

These documents are not intended to prescribe "the right way" to build software, nor are they a catalogue of best practices.

Instead, they describe the reasoning behind recurring design decisions, architectural trade-offs, and development workflows that have consistently informed my projects.

Many of these principles only became visible after reflecting on completed work. Rather than inventing a philosophy first and then trying to follow it, this repository attempts to document the principles that were already influencing engineering decisions.

As I gain experience, these documents may evolve. They should be read as a snapshot of my current engineering thinking rather than a fixed doctrine.

## Repository structure

* **principles.md**  
  Core engineering principles that guide design decisions.
* **learning.md**  
  How personal projects fit into my long-term learning strategy.
* **ai-collaboration.md**  
  How I use AI as an engineering collaborator while retaining responsibility for technical decisions.
* **project-template**/  
  A starting point for new repositories, including agent guidance and documentation templates that reflect the principles described here.
* **interaction-protocols**/  
  Concrete agent interaction protocols for specific use cases. These are protocols that are too small to warrant their own repositories.

## Relationship to project repositories

Individual projects should document the reasoning behind decisions that are specific to that project.

This repository documents the broader principles that influence those decisions across projects.
